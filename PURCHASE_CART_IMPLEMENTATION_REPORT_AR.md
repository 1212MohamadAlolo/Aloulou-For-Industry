# تقرير إضافة سلة المشتريات وفاتورة طلب الشراء

## ما تم تنفيذه

- تحويل نظام اختيار المنتجات السابق إلى **سلة مشتريات** تحتوي على الكميات.
- تغيير زر الرأس إلى **سلة المشتريات** مع عداد إجمالي القطع.
- عند الضغط على «أضف للسلة» تظهر نافذة لتحديد الكمية.
- بعد تأكيد الكمية تفتح السلة تلقائيًا وتعرض رسالة تشجع على متابعة التسوق أو إتمام الطلب.
- إضافة أزرار زيادة ونقصان وحذف لكل منتج داخل السلة.
- حفظ السلة محليًا في المتصفح باستخدام `localStorage`.
- ترحيل المنتجات القديمة المحفوظة في قائمة طلب السعر إلى السلة بكمية 1.
- إضافة زر كبير وواضح: **إتمام الطلب عبر WhatsApp**.
- إضافة نافذة تأكيد تحتوي على اسم العميل وملاحظات اختيارية.

## الفاتورة الذاتية

عند إتمام الطلب ينشئ الموقع تلقائيًا:

1. ملف PDF بمقاس A4 وهوية علولو للصناعة.
2. ملف HTML مستقل للطلب.
3. رقم طلب وتاريخ إنشاء.
4. جدول يحتوي على اسم كل منتج والموديل والكمية المطلوبة.
5. رأس وتذييل بهوية الموقع ورقم قسم المبيعات.
6. تنبيه أن الوثيقة طلب شراء أولي وأن السعر والتوفر يؤكدان عبر WhatsApp.

## آلية WhatsApp

- على الهواتف التي تدعم مشاركة الملفات من المتصفح، يفتح النظام نافذة المشاركة الأصلية مع ملف PDF والرسالة.
- يختار المستخدم تطبيق WhatsApp ثم جهة الاتصال المطلوبة.
- إذا كان المتصفح لا يدعم مشاركة الملفات، يتم تنزيل PDF وHTML وفتح محادثة قسم المبيعات برسالة الطلب الجاهزة.

## قيد تقني مهم

لا يستطيع موقع GitHub Pages الثابت إرفاق ملف PDF وإرساله تلقائيًا إلى جهة اتصال WhatsApp محددة دون تفاعل المستخدم. الإرسال التلقائي للوثائق يحتاج خادمًا وربطًا رسميًا مع WhatsApp Business Cloud API.
